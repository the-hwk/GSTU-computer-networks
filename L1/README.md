### Краткие теоретические сведения

#### Компьютерная сеть
**Компьютерная сеть** - это группа взаимосвязанных вычислительных устройств (компьютеры, серверы, принтеры и т.д.), которые могут обмениваться данными между собой. Основные компоненты сети:
* **Узлы (Nodes):** устройства, которые отправляют, получают или пересылают данные (компьютеры, маршрутизаторы, коммутаторы)
* **Среда передачи данных (Transmission Medium):** физический канал, по которому передаются данные (например, медный кабель, оптоволокно, радиоволны Wi-Fi)
* **Сетевые протоколы (Network Protocols):** набор правил и стандартов, которые определяют, как узлы обмениваются информацией. TCP/IP (Transmission Control Protocol/Internet Protocol) является основным стеком протоколов для большинства современных сетей, включая Интернет

Для успешного обмена данными в сети каждое устройство должно иметь два типа адресов: **IP-адрес** и **MAC-адрес**:
- **MAC-адрес (Media Access Control Address)** является физическим (аппаратным) адресом. Он зашит в микросхему сетевого адаптера производителем и уникален в мировом масштабе. MAC-адрес оперирует на канальном уровне (Layer 2) модели OSI и используется для доставки данных внутри одного сегмента локальной сети (LAN)
- **IP-адрес (Internet Protocol Address)** является логическим адресом. Он назначается администратором или DHCP-сервером и используется для маршрутизации пакетов между различными сетями (например, в Интернете). IP-адрес оперирует на сетевом уровне (Layer 3) модели OSI и состоит из адреса подсети и адреса узла. IP-адрес необходим для того, чтобы пакеты могли найти путь к удаленной сети

Важной частью IP-адреса явялется **маска подсети**. Она необходима для определения **адреса подсети** и **адреса узла**. - Маска `255.255.255.0` в двоичном виде выглядит как `11111111.11111111.11111111.00000000`. Те части маски, которые содержат `255` (единицы), соответствуют **адресу подсети**. Та часть, которая содержит нули, соответствует **адресу узла (компьютера)**.

Например, для IP-адреса `192.168.10.5` и маски `255.255.255.0`:
- адрес сети: `192.168.10` (первые три октета)
- адрес узла: `5` (последний октет)

#### Системные утилиты операционных систем для мониторинга сети
Современные операционные системы (ОС), такие как Windows, Linux или macOS, содержат встроенные службы и программы, которые управляют сетевым взаимодействием и позволяют администраторам настраивать, диагностировать и мониторить работу сети.

Основные системные утилиты командной строки:
- `ipconfig` - отображает текущие параметры сетевых интерфейсов. Показывает IP-адрес устройста, маску подсети, основной шлюз, MAC-адрес сетевого адаптера
- `ping` - проверяет связность между двумя узлами в сети, используя протокол ICMP (Internet Control Message Protocol). Показывает, доступен ли удаленный узел, и время задержки (latency) в миллисекундах
- `arp` - управление таблицей ARP (ARP cache), которая сопоставляет IP-адреса с MAC-адресами в пределах локальной сети. Когда узел знает IP-адрес другого узла в той же сети, но не знает его MAC-адрес, он использует ARP для его определения, чтобы доставить кадр данных
- `tracert` (Windows), `traceroute` (Linux/macOS) - отслеживание маршрута следования пакетов от исходного узла до указанного узла. Показывает список маршрутизаторов, через которые проходит пакет, и время отклика каждого из них. Помогает диагностировать задержки и определить, где произошла ошибка на пути к удаленному ресурсу

#### Другие важные службы в компьютерных сетях
* **DHCP (Dynamic Host Configuration Protocol):** служба, которая автоматически назначает IP-адреса, маски подсети и адреса шлюзов новым устройствам в сети
* **DNS (Domain Name System):** служба, которая преобразует доменные имена (например, `google.com`) в IP-адреса (например, `142.250.66.110`)
* **Firewall (Брандмауэр/Межсетевой экран):** cлужба, которая контролирует входящий и исходящий сетевой трафик на основе установленных правил безопасности, защищая систему от несанкционированного доступа

### Задание

#### 1. Определение сетевых параметров
1. Запустите командную строку (Win+R -> cmd).
2. Введите команду `ipconfig /all`.

В отчете зафиксировать:
- свой IPv4-адрес
- маску подсети (Subnet Mask)
- основной шлюз (Default Gateway)
- физический адрес (MAC-адрес) сетевого адаптера
- адрес DNS-сервера

#### 2. Работа с ARP-таблицей
1. После выполнения команды `ping` к соседу, введите команду `arp -a`.
2. Найдите в списке IP-адрес соседа.

В отчете: 
- выпишите соответствующий ему MAC-адрес и тип записи (динамический/статический)

#### 3. Проверка связности
1. Выясните IP-адрес компьютера соседа (спросите у него).
2. Выполните команду `ping [IP-адрес соседа]`.
3. Проверьте работу стека TCP/IP на собственной карте, выполнив команду `ping 127.0.0.1`.
4. Попробуйте пропинговать известный внешний ресурс, например `ping google.com`.

В отчете: 
- скриншоты с результатами выполнения команды `ping` на каждом этапе

#### 4. Трассировка маршрута
1. Введите команду `tracert yandex.by` (или любой другой внешний сайт). Дождитесь завершения трассировки.

В отчете:
- скриншот с результатом выполнения команды `tracert`
- cколько "хопов" (прыжков) до конечного узла
